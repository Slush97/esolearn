| ID | Issue | File:Line | Recommendation |

| M1 | **Bar chart not always zero-anchored** — positive-only data nices to e.g. [40,100], bars appear to start at bottom but axis starts at 40. Classic truncated bar chart. | mod.rs:222-234, mark_gen.rs:303-307 | Force `y_min = y_min.min(0.0)` for bar/area marks |

## How This Audit Was Conducted (Methodology for Reuse)

This audit used 4 complementary AI strategies run in parallel:

### Strategy 1: Spec-Driven Compliance
- **Input**: `AUTOFORMAT_RULES.md` + all `compile/*.rs` + `new_theme.rs`
- **Prompt**: "For each spec rule, verify the code matches. Report PASS/FAIL/MISSING with line numbers."
- **Yield**: 28 spec violations found. Highest signal-to-noise of all strategies.

### Strategy 2: Best-Practices Reference Comparison
- **Input**: `mark_gen.rs`, `axis_gen.rs`, `legend_gen.rs`, `layout.rs`, `new_theme.rs`
- **Prompt**: "Compare against Vega-Lite/ggplot2/matplotlib defaults. Identify misleading > broken > ugly > suboptimal."
- **Yield**: 17 issues, including all 5 MISLEADING findings. Best at catching semantic correctness issues.

### Strategy 3: API Design Review
- **Input**: `express/mod.rs`, `grammar/*.rs`, `lib.rs`, `error.rs`
- **Prompt**: "Review for naming consistency, type safety, ergonomics, feature completeness."
- **Yield**: 28 issues. Unique contribution: found panic path, semver risks, parameter order inconsistencies.

### Strategy 4: Visual Edge-Case Testing
- **Process**: Write edge-case example program → generate SVGs → inspect raw SVG markup
- **Edge cases tested**: single point, single bar, all-negative bars, mixed +/- bars, 1000-point scatter, long labels, large/small numbers, narrow/wide charts, dark theme, flat line, many pie slices, heatmap
- **Yield**: 11 visual issues. **Only strategy that caught the single-bar width bug and negative-bar visual confusion.** Essential complement to code review.

### Recommendation for Ongoing Use
Run all 4 strategies after each major feature addition. Strategy 4 (visual testing) should be automated into CI as a regression check — generate reference SVGs and diff against known-good baselines.
